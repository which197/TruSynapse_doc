神经网络数据组织与压缩
===============================


数据类型
-------------------------

- 神经元状态（neuron state）
    - 记录神经元膜电位及其随时间的变化（例如用于仿真时的内部变量）。
- 连接关系（connectivity / synapses）
    - 描述神经元之间的突触连接，主要包含权重信息和连接拓扑。
- 输入 spike
    - 输入到网络的事件流（时间序列形式的脉冲/ spike）。
- 输出 spike
    - 网络产生的事件流（作为仿真或推理的输出）。

存储与压缩
----------------

- 神经元状态
    - 每个神经元有唯一状态，适合顺序（数组）存储，便于按索引快速访问。
- 输入/输出 spike
    - 通常为时间序列，也可按时间或事件索引顺序存储（顺序存储或事件列表）。
- 连接关系（稀疏性与查找）
    - 大规模网络中连接关系高度稀疏，直接使用稠密矩阵会浪费空间。
    - HPC 常见的稀疏存储（例如三元组（``i, j, value``））在查询和随机访问上开销较大，因此选择合适的存储格式对压缩率和访存效率至关重要。

传统表示与 TruSynapse 方案
-------------------------------

传统上，连接信息常用两张表来表示：

- 权重表: ``Weight<i,j>`` 表示从 ``i`` 到 ``j`` 的权重值。
- 连接表: ``C<i,j>`` 为 ``1`` 表示存在连接， ``0`` 表示无连接。

二者关系: 在权重表中非零的位置，连接表对应位置必然为 ``1``。

TruSynapse的做法是将两张表合并为一张表，在连接记录中直接存储权重信息。

上述压缩后的连接关系表 :math:`C` 需要拆分到多个NFU中:

- 每个 NFU 只存储子表 :math:`C[i]` ( :math:`i` 为 GNC的ID)。所有子表的并集构成全局表 :math:`C`, :math:`C = \bigcup_{i \in I} C[i]`。
- 各子表不同， :math:`\forall i,j(i\neq j), C[i]\cap C[j] = \emptyset`
